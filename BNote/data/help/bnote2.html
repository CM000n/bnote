Das aktuelle Release dieser Software ist Version 2.4.
Auf GitHub sind genaue Angaben zu Features und Bugs die mit dieser Version behoben sind vorhanden. Hier jedoch eine kleine Übersicht mit den wichtigsten Änderungen:

Umbenennung in BNote

Mit dem aktuellen Release wurde die Umbenennung des System von BlueNote in BNote vollständig abgeschlossen. Dies war aus markenrechtlichen Gründen notwendig.

Probenverwaltung

In der aktuellen Version ist es nun möglich mehrere Proben auf einmal zu erstellen. So können beispielsweise zu einem festgeleten Zeitpunkt jede Woche in den nächsten 10 Wochen die Proben automatisch erstellt werden.
Außerdem gibt es die Möglichkeit Probenphasen anzulegen. Eine Probenphase enthält mehrere Proben, Konzerte und Kontakte.
Als Detailverbesserung wird nun zu jeder Probe angezeigt, welche Instrumente anwesend sein werden.

Styling

Insbesondere die Startseite nach dem Login wurde umgestaltet. Der neue Style ist an typischen Dashboards ausgerichtet und soll dem Benutzer einen schnellen Überblick geben.

Konfiguration

Dem Bandleiter ist es nun möglich die Benutzer zu bearbeiten und grundsätzliche Einstellungen am System vorzunehmen. Hierfür wurde das neue Modul "Konfiguration" erstellt.

Kalender Abonnement

Der Kalender kann nun immer aktuell als iCalendar in den persönlichen Kalender, z.B: bei Google integriert werden. Hierzu wurde auf der Startseite der neue Knopf "Kalender abonnieren" hinzugefügt.

Aufgaben

Bandleiter können nun Aufgaben erstellen und diese einzelnen Mitspielern zuweisen. Sind diese mit einer Aufgabe fertig, so kann diese als fertig gekennzeichnet werden.

Nachrichten

Um allen Mitspielern etwas mitzuteilen, kann man das Nachrichtenmodul verwenden. Die Nachricht(en) werden dann auf der Startseite aller angezeigt.

Share

Die Dateiverwaltung wurde komplett überarbeitet. Hier wurden ein gruppenbasiertes Zugriffskonzept, Gruppen- und Benutzerordner eingeführt.

Fehlerbehebung

Außer den zuvor genannten Änderungen wurden von Testern und Benutzern gemeldete Fehler behoben.

Genauere Hinweise zu den Änderungen und Neuerungen der einzelnen Versionen finden sich beim jeweiligen Update in der Datei release_notes.txt.
